# 招聘考题：PPT 一套生成器

## 任务

写一个程序，输入是一个 JSON，输出是一套 25-30 张的 .pptx 文件。

**核心要求**：输出是一**套** 风格一致的PPT，不是一**堆** PPT——25-30 张幻灯片在内容上是连贯的整体（有开头、有展开、有收尾、有内在叙事线），不是 30 张互相独立的卡片。

## 输入格式

```json
{
  "topic": "主题",
  "brief": "简介（≤500 字）",
  "audience": "目标受众"
}
```

## 输出

- 一套 25-30 张的标准 .pptx 文件（PowerPoint / Keynote 能正常打开）
- **单条命令吃 JSON 吐 .pptx

## 考核维度

三个考核点：**美观度**、**LLM 成本**、**生成速度**

## 工具与限制

- 可以自由选择任何 LLM、任何编程语言、任何工具链
- **不准**直接使用 Gamma / Tome / Beautiful.ai 等"一键生成 PPT"的 SaaS 产品
- 鼓励使用 Claude / Codex / Cursor / Copilot 等 AI 编码助手——这是题目设计的一部分

## 时间盒

**3 天（72 小时）**，从拿到题面起算。超时不加分。

## 交付物

提交一个 Git 仓库（GitHub / GitLab 私有仓即可，加我们为协作者），包含：

### 1. 代码

- 完整可运行的源代码
- `README.md`：怎么装依赖、怎么配 API key、怎么跑一个 demo

### 2. 决策日志（`DESIGN.md`，2-3 页）

必须包含以下小节：

- **架构图 + 数据流**：整套系统怎么跑的
- **模型选型**：用了哪些模型、为什么选这些、有没有比较过其他
- **风格一致性怎么保的**：这是核心难点，详细写
- **多样性怎么保的**：不同输入的产出不能像同一个模板填空
- **成本与时延实测**：5 套 demo 的实测数据表格
- **踩坑和取舍**：哪些方案试过被否、为什么
- **AI 协作复盘**：这一段很重要，必写——
  - 哪些决策是 AI 提的、你照做了
  - 哪些是 AI 提的、你推翻了，为什么
  - 有没有出现"AI 跑偏你把它拽回来"的场景，举一两个具体例子
  - 你对 AI 的输出做了哪些核验/兜底

**没有这一段，决策日志维度直接扣一半。** 我们想看的是"用 AI 加速但保持判断"的人，不是"被 AI 牵着走"的人。

### 3. 5 套 demo（两版）

用下方公开开发集的 5 个输入跑出来的 .pptx 文件，外加两版每套的成本/时延实测数据。

## 公开开发集（5 个输入，公开调试用）

| # | topic | brief | audience |
|---|-------|-------|---------|
| 1 | Python 入门 30 分钟 | 帮零基础的人理解什么是变量、循环、函数、列表、字典，并能照着写出几行能跑的代码 | 编程零基础 |
| 2 | 2025 我的年度复盘 | 这一年我做的几件主要的事、踩过的几个坑、明年想试的方向 | 朋友圈分享 |
| 3 | 如何挑选一款适合自己的咖啡豆 | 从烘焙度、产地、处理法、风味描述讲起，给一个挑豆决策框架 | 咖啡新手 |
| 4 | 给老板讲清楚为什么我们应该用 Rust 重写订单系统 | 现在用 Python 写的订单系统在大促时频繁超时，影响下单。希望说服一个非技术背景的 CEO 同意立项 | 非技术 CEO |
| 5 | 周末两天玩遍京都 | 周六上午到周日晚上，预算人均 3000，希望涵盖经典景点和一两个小众体验 | 第一次去日本的游客 |

### 最大化美观度版

旨在考察美观度最好能做到什么程度

成本限制：单份消耗金额小于10美金

速度限制：单份生成时间小于30分钟

### 美观度-成本-速度 Trade-off版

自行对美观度-程度-速度 做trade off



**要求**：两版各提交5个主题的ppt

## 关于这道题

这是一道**开放题**——用 AI 协作怎么都能跑通，区别只在做到 1 分还是 100 分。

我们想看的是：**给你一个真实业务问题、一些硬约束（成本/时延）、一个时间盒，你能不能在 AI 加持下交付一个既能跑、又便宜、又快、又好看的成品。** 这正是入职后日常工作的样子。

祝你做出让自己满意的版本。
